# 호흡 세기 (Breath Counting, Levinson 2014) — 설계·작업로그

실천(practice) 계열 **1번째** 과제. 인지 9 / 지각 10 / 마음챙김 4 에 이어 신설한 계열.
청소년·성인 2앱이 `breath-counting-common/breath-counting.js` 를 공유(`startBreathCounting(opts)`).

## 이 계열이 다른 이유 — 측정이 아니라 체험
지금까지 23과제+4데모는 전부 "무언가를 측정하고 결과를 보여주는" 성격이었다. 실천 계열은
근본적으로 다르다 — **측정이 아니라 훈련/개입 체험**이다. 그래서 엔진(`core/engine.js`)이 전제하는
"시행-채점" 구조가 맞지 않아 `runTask` 를 쓰지 않는다. 지각 그룹D 독립-데모 패턴 그대로:
`core/i18n.js`(언어감지·전환·푸터)만 재사용하고 자체 셸을 그린다. **엔진 코어는 계열색 한 줄만 수정.**

## 계열색 (엔진 유일한 수정)
`FAMILY_COLORS.practice = '#4A6B4D'` (세이지그린). 대비 확인:
- **성인 `#4A6B4D`**: 흰 텍스트 대비 **6.0:1** (WCAG AA 통과). 계열 기준값이자 성인 앱 accent.
- **청소년 `#56805A`**: 흰 텍스트 대비 **4.6:1** (AA 통과). 같은 세이지 계열의 밝은 톤(명도만 위로).
  더 밝게 가면 흰 글자 대비가 4.5:1 아래로 떨어져 이 값이 사실상 상한.
실천 계열은 runTask 를 안 쓰는 체험 도구라, 각 앱 셸이 자기 `accent` 를 직접 들고 `--bc-accent` 로
주입한다(FAMILY_COLORS 는 계열 기준·문서화용으로 등록).

## ★ 시각 예외 — 왜 이번엔 일러스트(싱잉볼)를 썼나 (문서화 필수 항목)
지각 계열의 "**자극은 무채색, accent 는 UI 전용**" 원칙은 **판단 대상 자극**(피험자가 보고 판정하는
그 자극)에 적용되던 규칙이다. **이 과제엔 판단 대상 자극이 없다** — 체험 도구이지 측정 도구가
아니기 때문이다. 그래서 싱잉볼 그림을 쓰는 것은 **원칙 위반이 아니라 적용 범위 밖**이다(측정이
아니라 체험이라 시각적 절제의 기준이 다르게 적용됨). 잔상(afterimage)이 "색 자체가 자극"이라
색 원칙의 예외였던 것과는 또 다른 결 — 여긴 아예 판정 대상이 없어 규칙이 미적용된다.

- 싱잉볼은 **장식이 아니라 '호흡 표시 기능'**: 숨을 셀 때마다(왼쪽/오른쪽 카운트 버튼) 볼 입구에서
  물결(동심원)이 은은하게 퍼진다. `rippleAt()` 이 누를 때마다 `<circle class="bc-ripple">` 를 하나씩
  append(각 물결이 자기 element 라 빠른 연타도 겹쳐 보임), 애니메이션 끝나면 스스로 제거.
- 그림체: **사람 형태 아님**(볼), 선화·평면, 색은 accent 계열에서 자유, 외부 이미지 없이 **SVG 직접 렌더**.

## 과제 설계 (Levinson 2014)
- 숨쉬며 **속으로 1~9까지** 셈(화면 조작 없이 마음속으로만).
- **한 사이클(1~9) 완료 = 화면 아무 곳이나 톡**(큰 터치존). 정확한 버튼을 찾을 필요 없이 눈 감고도 가능.
  탭할 때마다 사이클 수 +1, 싱잉볼 중심에서 물결.
- 세다 놓쳤음을 스스로 알아차리면 **'놓쳤어요'(self-caught) 버튼**(하단 별도, 작게) → 그 시점만 기록.
  **완료한 사이클 수는 깎지 않음**(완료한 사이클은 완료된 것; '진행 중이던' 한 사이클만 사용자가 마음속 리셋).
- 세션 길이 **3/5/10분** 자기 선택. 시간 다 되면 자동 종료.
- **정답/오답 판정 없음.** 탭이 실제 호흡과 맞는지 앱은 알 방법이 없다(자기보고) — 판정 안 함.

### ★ 조작 변경 이력 (v2 — 명상 흐름 우선)
초기 v1 은 **1~8 왼쪽 버튼 / 9 오른쪽 버튼**(매 호흡마다 정확한 버튼 누름)이었으나, "매번 정확한 버튼을
찾아 누르는 게 명상 흐름을 깬다"는 피드백으로 v2 로 바꿨다: **왼쪽(1~8) 버튼 제거, 1~8은 마음속으로만
세고, 한 사이클 완료만 화면 전체 터치존으로.** 조작이 세션당 '사이클마다 한 번, 아무 곳이나'로 단순화됨.
화면 숫자도 '1~9 카운트' → **'완료한 사이클 수'**로 바뀜. 큰 터치존은 `<button>`(a11y aria-label 필수:
"한 순환을 마쳤으면 여기를 눌러주세요"), `min-height:58dvh`로 카드 대부분을 덮음. '놓쳤어요'는 밖의 별도
버튼(전파 차단 `stopPropagation`)이라 오집계 없음. 물결은 터치 위치와 무관하게 항상 싱잉볼 중심(120,78).

## 화면 (4언어)
1) 안내 → 2) 세션 길이 선택 → 3) 세션(큰 터치존 안에 싱잉볼·타이머·사이클 수·안내 + 하단 '놓쳤어요') →
4) **완료 확인**(‘결과’ 아님). 완료 화면: 총 시간·**완료한 사이클 수**·'놓쳤어요' 횟수만 **사실로** 표시.
좋다/나쁘다/정상/비정상, '충동·집중력' 어휘 **금지**(SART·주의순간멈춤 원칙 그대로 — '놓침·알아차림' 사용).
세션 내 시점을 **2-레인 스파크라인**(중앙선 위=사이클 완료 muted, 아래='놓쳤어요' accent)으로 표시(있을 때만,
범례 2개). **세션 간 비교·최근 날짜 기록 없음.**

## 실천 계열 원칙 — localStorage 안 씀(스트릭 없음)
언어 선택(`cog:lang`, i18n 공용)을 빼면 **아무것도 저장하지 않는다**. 세션 기록·비교·스트릭 없음.
음소거 상태도 저장 안 함 — 매 세션 기본값(소리 켜짐)으로 시작.

## 싱잉볼 소리 (순수 부가 요소)
- **시점**: 세션 시작(길이 버튼 클릭) **그 순간 딱 1번**. 사이클 탭·중간엔 소리 없음(물결만).
- **합성**: 외부 파일 없이 Web Audio 로 합성. 기본음 220Hz + 비조화 배음 607·1188Hz(배수 ≈1/2.76/5.40)
  사인 오실레이터 3개, 각기 20ms 어택 → 5/4/3초 지수 감쇠. `playBowl()` 참고.
- **자동재생 정책**: `AudioContext` 를 반드시 그 클릭 핸들러 **안에서** 생성(모듈 로드 시 미리 안 만듦).
- **★ resume 레이스 방어(v2)**: `playBowl` 은 `async` 로, `state==='suspended'` 면 **`await audioCtx.resume()`
  로 완료를 기다린 뒤** `now = ctx.currentTime` 를 읽어 스케줄한다(생성은 여전히 제스처 동기 실행이라
  자동재생 정책은 충족). resume 전에 스케줄하면 일부 브라우저에서 첫 음이 씹힐 수 있어 방어. `catch(e)` 는
  **조용히 삼키지 않고 `console.error`** 로 남긴다(F12 콘솔 확인용). 참고: 헤드리스 Chrome 은 오디오 장치가
  없어 `currentTime` 이 0에 머물고 소리를 실제로 못 낸다 → 소리 자체는 실제 브라우저에서만 검증 가능(그래프
  계측으로 오실레이터·주파수·엔벨로프만 자동 확인).
- **음소거 토글**: 세션 화면 헤더(🔔/🔇). 기본 켜짐, 누르면 울리는 볼을 짧게 페이드아웃(`silenceBowl()`).

## 세션 화면 언어바 숨김 (측정 과제와 다른 선택)
측정 과제(엔진)는 시행 도중에도 언어바를 항상 띄운다(setLang 이 자극·진행표시만 갱신, 전역 시계
없음). 이 과제는 **세션(체험) 화면에서만 언어바를 숨긴다** — 집중 방해 최소화 + 세션 도중 언어 전환에
따른 재렌더 경로 자체를 없애기 위함. (그 재렌더가 타이머를 다시 찍어 '함께한 시간'이 어긋나던 버그를
근본 차단. 더해 `sessionStart` 는 세션 진입 시 한 번만 찍고 재렌더에서 다시 찍지 않도록 방어.)
인트로·길이·완료 화면에서는 언어바 정상 노출.
