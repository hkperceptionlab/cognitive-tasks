# 가이드 명상 타이머 (Guided sitting timer) — 설계·작업로그

실천(practice) 계열 **4번째·마지막** 과제(1=호흡세기, 2=바디스캔, 3=자애·감사). 청소년·성인 2앱이
`guided-timer-common/guided-timer.js` 를 공유(`startGuidedTimer({id, accent, scale, tone, scene})`).
UI 표시명 한국어 "고요한 시간"(폴더/코드 영문명 `guided-timer`).

## 앞의 세 과제와 같은 기준 (재확인)
runTask 안 씀, `core/i18n.js` 만 재사용·자체 셸, 판정 어휘 금지, '결과' 아니라 '완료 확인',
**localStorage 아무것도 안 씀**(언어선택 `cog:lang` 제외), 계열색 practice 재사용(성인 `#4A6B4D` /
청소년 `#56805A`, 셸이 `--gt-accent` 주입). **엔진 코어는 이번에도 무수정**.

## ★ 넷 중 가장 단순한 구조
특정 기법(세기·훑기·마음 보내기) 없이 "정해진 시간 동안 안내와 함께 조용히 앉아 있는" 범용 타이머.
자기보고·부위 순서·대상 선택 없음. 세션 중 사용자 조작은 **음소거 토글뿐**(연꽃의 건너뛰기 같은 것도 없음).
완료 화면은 **"함께한 시간"만**(다른 지표·그림 없음).

## ★ 개정 이력
최초 설계는 숫자 타이머 + 원형 프로그레스(그림 없음)였다. 논의 중 "시간이 흐르며 완성되는 자연 풍경"으로
진행 표시 자체를 바꾸기로 확정 — 기계적 게이지 대신 해가 뜨는 것·별이 켜지는 것 같은 자연스러운 시간 감각.
**숫자·원형 프로그레스는 완전히 제거**(화면엔 그림만).

## ★ 시각 예외 — 네 번째 근거 (DESIGN 문서화 필수)
호흡세기 싱잉볼='장식이지만 호흡표시 기능', 바디스캔 실루엣='안내 자체의 기능', 연꽃='단계별 진행의
시각적 서사'. 이번은 **또 다른 네 번째 근거**:

> 이 그림 = 숫자·게이지 같은 기계적 진행 표시를 완전히 대신하는, **자연의 시간 감각으로 표현한 진행
> 지표**. 연꽃처럼 '단계'가 있어 단계마다 바뀌는 게 아니라, **정해진 단계 없이 경과 비율(elapsed/total)에
> 매끄럽게 연동되는 '연속적' 변화**라는 점이 연꽃과의 핵심 차이(그래서 바디스캔·연꽃의 'TTS onend와
> 최소시간 중 더 늦은 쪽' 진행 규칙이 여기선 아예 불필요 — 순수 타이머 기반).

- **무채색 원칙 예외 재확인**: 하늘·해·별은 실채색(잔상 데모와 같은 결). 그림이 표현하려는 것(새벽·밤하늘)이
  무채색으로는 성립 안 되므로 당연한 예외. accent(계열색)는 UI(버튼·언어탭)에만.

## ★ 처음으로 성인·청소년 그림 '내용 자체'가 다름 (근거 문서화 필수)
앞의 세 과제(호흡세기·바디스캔·연꽃)는 두 폴더가 **계열색 명도만** 다르고 그림 소재는 완전히 같았다.
이번엔 소재 자체가 다르다. `scene` 옵션으로 분기:

- **성인(`scene:'sunrise'`)**: 일출. 어두운 남색 하늘 → 새벽 보라 → 주황·분홍 → 밝은 하늘색으로 하늘
  그라데이션이 보간되고, 해(원)가 수평선 아래에서 경과 비율에 비례해 떠오른다. 차분하고 절제된 톤.
- **청소년(`scene:'stars'`)**: 밤하늘(고정 남색~보라)에 별 9개가 하나씩 켜짐(회색 점 → 노란빛, 은은한
  반짝임). 발견하는 재미, 성인판보다 조금 더 생동감. 초승달·언덕 실루엣은 고정 장식.

**이 차이는 "성인은 차분, 청소년은 재미"라는 취향이 아니라, 명상이라는 낯설 수 있는 경험에 진입하는
정서적 톤을 연령대에 맞춘 것**이 근거. → 다음에 또 연령별 그림을 다르게 할 일이 생기면 이 사례를 참고
삼되, 매번 "그냥 다르게 하고 싶어서"가 아니라 각각 이유를 새로 세울 것.

## 진행 로직 (단계 없음)
`startSessionTimer()`의 rAF 루프가 매 프레임 `p = elapsed / sessionMs`(0~1)를 계산해 `updateScene(p)` 호출.
그림 요소는 세션 진입 시 `cacheSceneRefs()`로 한 번만 캐시(매 프레임 querySelector 회피).
- **일출**: `skyAt(p)`가 `SKY_KF`(4 키프레임)를 색 보간(`lerpColor`) → 하늘 3-stop 그라데이션·해 색·땅 색
  갱신, 해 `cy`를 `SUN_Y0(250, 수평선 아래)→SUN_Y1(78)`로 선형 이동(땅 rect 가 아래를 가려 떠오르는 느낌).
- **별**: 별 i 는 `p >= (i+0.5)/N`일 때 `.on`(연꽃 꽃잎 로직과 유사하되 순수 타이머라 더 단순, 누적).
- 세부 수치(별 개수 9·해 이동 범위·키프레임 색)는 코드에서 시각적으로 다듬은 값 — 스크린샷 눈확인 필요.

## ★ 소리
- **시작·끝 종소리**: 호흡세기 `playBowl`의 Web Audio 합성 재사용(`playBell(base)`, 비조화 배음 1/2.76/5.40).
  **음높이로 구분**(시작 `START_FREQ=220`, 끝 `END_FREQ=174.61`, 더 낮게). AudioContext 는 길이선택 클릭
  핸들러 안에서 최초 생성(`playBell(START_FREQ)` — 자동재생 정책 회피), 끝 종소리는 자동 종료 시점에 울림
  (이미 생성·resume 된 컨텍스트라 재생 가능).
- **음성 리마인더**: 약 90초 간격(`R_INTERVAL`, 첫 리마인더 `R_FIRST=20s`), 6개 문구(`r0..r5`) 순환.
  `speechSynthesis`로 읽고 화면 하단 `.gt-reminder`에 잠깐 표시 후 페이드아웃(`R_SHOW_MS=4.5s`). **단계 전환이
  아니라 순수 타이머 기반**이라 진행을 막지 않으므로 바디스캔/연꽃급 onend 안전장치 불필요. 종료 직전
  `R_GUARD=15s` 구간엔 리마인더를 띄우지 않음(끝 종소리와 분리). 미지원·해당언어 음성 없음·음소거면 조용히
  텍스트만(정상 폴백).
- **음소거 토글**(🔔/🔇): 종소리·리마인더 음성 끔(그림·텍스트는 계속). 누르면 `silenceBell()`+`speechSynthesis.cancel()`.
- **기법 없는 과제라 리마인더는 특정 방법을 지시하지 않음** — "그대로 머물러 보세요" 같은 부드러운 상기만.
- **음질은 사용자가 실제 브라우저서 직접 확인**(소리 자체는 헤드리스 검증 불가 — 호흡세기·바디스캔과 동일).

## 화면 (4언어)
1) 안내(방법 없음·그림이 시간에 따라 완성·시작/끝 종소리·판정 없음) → 2) 세션 길이 선택(5/10/15분, 이
클릭이 시작 종소리 트리거) → 3) 세션(그림 + 리마인더 텍스트, **숫자·게이지 없음**, 음소거 토글, 언어바
세션중 숨김) → 4) **완료 확인**(함께한 시간만, 그림 없음 — 세션 전용).

## QA (판정·자기보고 지표 없는 체험 — 그룹D 데모식 스모크)
`qa/check.mjs` kind='guided-timer'. `data-scene` 속성으로 분기:
- 공통: 로드·안내문·언어4·길이3·음소거토글·완료화면(함께한 시간만·그림 없음)·**입력 UI 부재(전 구간
  감시)**·리마인더 텍스트 최소 1회 표시됐다 사라짐·JS에러0.
- 성인(sunrise): 진행에 따라 해 `cy` 감소(초기 vs 중간)·하늘 stop-color 변화.
- 청소년(stars): `.gt-star.on` 개수가 시간 경과에 따라 누적 증가, 최종 9개 모두 켜짐.
`?qa=1` 이면 세션 3초·리마인더 간격 0.7초로 축약(바디스캔 TTS_CAP 축약과 같은 접근). 오디오(종소리)는
헤드리스로 "울렸는지"까지 검증 불가 — 실제 청취·그림의 '느낌'은 스크린샷 눈확인 + 재생 확인으로 분리.
