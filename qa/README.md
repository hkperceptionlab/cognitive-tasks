# qa — 회귀 점검

엔진(`core/`)이나 과제 정의를 고칠 때마다 돌려서 4개 앱이 안 깨졌는지 자동 확인한다.

```bash
cd qa
npm install    # 최초 1회 (puppeteer-core)
npm run check
```

설치된 Chrome 을 헤드리스로 띄워 **스트룹 청소년/성인, Go/No-go 청소년/성인**을 실제로 플레이한다.
봇은 정답만 낸다: 스트룹은 잉크색 버튼, Go/No-go 는 원에만 응답(사각형은 참음).

각 앱마다 두 회차를 돌리며 확인하는 것:

| 점검 | 방법 |
| --- | --- |
| **결과 화면 도달** | 연습→본시행을 끝까지 자동 플레이해 요약(`.summary`)이 뜨는가 |
| **JS 에러 없음** | pageerror·console.error 가 0 인가 |
| **정답 판정 = 정확도 100%** | 정답만 냈으니 세션 `acc` 가 1 이어야 함. (Go/No-go 의 'No-go=안 누름이 정답' 훅이 깨지면 acc 가 떨어져 실패) |
| **다른 언어 기록이 걸러짐** | 회차 A(ko)·회차 B(en) 을 남겨두고, en 결과에서 그래프가 en 세션만 그리고 ko 는 '다른 조건 N회 미표시'로 안내되는가 (그래프 점 수 = 시리즈 수 × 내조건 세션 수 로 교차검증) |

전부 통과하면 종료 코드 0, 하나라도 실패하면 1 (`echo $?` 로 확인).
