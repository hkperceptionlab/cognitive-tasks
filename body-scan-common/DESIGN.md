# 바디스캔 (Body Scan Meditation) — 설계·작업로그

실천(practice) 계열 **2번째** 과제(1번째=호흡세기). 청소년·성인 2앱이
`body-scan-common/body-scan.js` 를 공유(`startBodyScan({id, accent, scale, tone})`).

## 호흡세기와 같은 기준 (재확인)
runTask 안 씀(엔진 시행-채점 구조 안 맞음), `core/i18n.js` 만 재사용·자체 셸, 판정 어휘 금지, '결과'가
아니라 '완료 확인', **localStorage 아무것도 안 씀**(언어선택 `cog:lang` 제외 — 스트릭·비교·기록 없음),
계열색 practice 재사용(성인 `#4A6B4D` 6.0:1 / 청소년 `#56805A` 4.6:1, 각 앱 셸이 `--bs-accent` 주입).
**엔진 코어는 이번엔 한 줄도 수정 안 함**(practice 색은 호흡세기 때 이미 등록됨).

## ★ 호흡세기와의 구조 차이 (핵심)
호흡세기는 자기보고(self-caught)가 핵심이라 self-paced였다. 바디스캔은 그런 자기보고 요소가 **없다** —
안내를 따라 시간에 맞춰 **자동으로** 다음 부위로 넘어가는 게 원래 방식. 그래서:
- **세션 중 사용자 조작은 음소거 토글뿐**(탭·버튼 없음).
- **완료 화면도 점수·자기보고 지표 없이** 총 시간 + 다녀온 부위 수(14, 고정)만 사실로 표시.
  억지로 점수화할 지표를 만들지 않음.

## ★ 시각 예외 — 몸 실루엣 (호흡세기 싱잉볼과 다른 근거, 문서화 필수)
호흡세기 싱잉볼은 "장식이지만 **호흡 표시 기능**이 있다"는 근거였다. 바디스캔 실루엣은 근거가 다르다:
**"몸의 어느 부위인지 보여주는 것이 안내 자체의 기능"** 이라 실루엣이 필요하다(측정 자극이 아니라 안내
도구). 지각 계열의 '자극 무채색' 원칙은 판단 대상 자극에 적용되던 것이고, 이 과제엔 판단 대상 자극이
없어(체험) 적용 범위 밖 — 호흡세기와 같은 결의 예외지만 **필요 근거는 서로 다름**(장식적 표시 기능 vs
안내의 본질적 기능).
- 정면을 향한 **아주 단순한 선화 실루엣**. 얼굴 표정 없음(빈 타원), **성별·나이·체형을 특정할 디테일
  전부 배제** — 캐릭터가 아니라 '부위 표시 다이어그램'.
- 13개 부위 그룹(`REGIONS`, `REGION_ORDER`)을 회색으로 조립(`.bsil-region` fill #e6e9e6). 지금 안내 중인
  부위 그룹만 accent 로 옅게(fill-opacity .5) 0.6s 페이드 전환. '온몸 전체'(`'ALL'`)는 모든 그룹 강조.
- SVG 로 직접 렌더(외부 이미지 없음). viewBox 200×470, 부위별 좌표는 코드 상수.

## 부위 순서 (자동 진행, 총 14단계)
발→종아리→무릎→허벅지→골반/엉덩이→배→가슴→손→팔→어깨→목→얼굴→정수리→온몸 전체(마무리).
`PARTS` 배열. 세션 길이 선택(5/10/15분) → **인트로(안착)·아웃트로(마무리) 예약 시간을 뺀 나머지를
14단계에 균등 분배**(`buildSegments`: perPart = (sessionMs − INTRO_MS − OUTRO_MS) / 14).
세그먼트 = [인트로, 14부위, 아웃트로]. 각 세그먼트에 최소 시간(minMs)과 진행바용 endFrac 부여.

## ★ 안내 전달 — TTS(목소리) + 텍스트 동시, 소리 껐다 켜기
- `speechSynthesis`(Web Speech API)로 각 부위 문구를 읽어주고, **동시에 화면에도 같은 문구를 항상 텍스트로
  표시**(`.bs-guide`, aria-live polite). TTS 음질·미지원 대비.
- **다음 부위로 넘어가는 시점 = TTS `onend` 와 부위별 최소 시간 중 '더 늦은 쪽'.** 구현: 세그먼트마다
  `cur.minDone`(minMs 타이머 발화 시 true)·`cur.ttsDone`(TTS 종료 콜백 시 true) 두 플래그, `maybeAdvance()`
  는 **둘 다 true 일 때만** 다음 세그먼트로. 최소 시간을 backstop 으로, TTS 가 더 길면 TTS 종료까지 대기.
- **폴백(에러 아님)**: TTS 미지원·해당 언어 음성 없음·음소거면 `speak()` 가 false 반환 → 그 즉시
  `cur.ttsDone = true` 로 두어 **최소 시간만으로 자동 진행**. `speak()` 는 try/catch 로 감싸 조용히 폴백
  (호흡세기 `playBowl` 패턴). 헤드리스 환경(음성 없음)이 이 폴백 경로를 그대로 검증한다.
- **첫 발화는 길이선택 버튼 클릭 핸들러 안에서 트리거**(render 의 session 분기에서 `playSegment(0)` 호출,
  이 호출 스택이 곧 클릭 제스처 — speechSynthesis 자동재생류 정책 회피). 호흡세기 싱잉볼과 같은 이유.
- **음성 선택**: `pickVoice()` 가 현재 UI 언어와 일치하는 `voice.lang`(ko→ko-KR 등, `startsWith(lang)`) 우선.
  없으면 텍스트만·소리 없이 최소 시간 진행(정상 동작). **rate 0.88**(명상 톤), pitch 기본.
- **음소거 토글**(🔔/🔇, 세션 헤더): 기본 켜짐, 세션마다 초기화(저장 안 함). 음소거 시 `speechSynthesis.cancel()`
  로 즉시 멈추고 `cur.ttsDone = true`+`maybeAdvance()` 로 텍스트+최소시간만으로 계속 진행(행 없음).
- **스테일 콜백 방지**: `segToken` 을 세그먼트마다 증가시키고 모든 콜백에서 토큰 일치 검사. `stopSession()`
  (완료·다시하기)은 토큰 증가+타이머 클리어+`speechSynthesis.cancel()`.
- ★ **음질은 실제 브라우저에서 사용자가 직접 듣고 최종 판단**(별도 모드 없음 — 별로면 음소거해 텍스트 전용).
  소리 자체는 헤드리스로 검증 불가.

## 화면 (4언어)
1) 안내(바디스캔 소개·누워도 됨·판정 없음) → 2) 세션 길이 선택(5/10/15분, 이 클릭이 첫 발화 트리거) →
3) 세션(실루엣[현재 부위 강조] + 안내 텍스트 + 얇은 진행바 + 음소거 토글, **언어바는 세션 중 숨김** —
호흡세기와 동일 근거) → 4) **완료 확인**(총 시간·다녀온 부위 수 14). 진행바는 각 세그먼트 minMs 동안
CSS transition 으로 채워지는 시간 진행 표시(점수 아님).
